# Docshub Phase 1 — MVP Implementation Plan

> **For Hermes:** Use Claude Code (`claude -p`) to implement each task sequentially.

**Goal:** Build a LAN document sharing hub where multiple users publish Markdown articles via CLI to a central Go server, with Docsify as the reading frontend.

**Architecture:** Go API server stores Markdown files in a directory structure, maintains a JSON index, and auto-generates `_sidebar.md` for Docsify. CLI client (also Go) handles publishing workflow with AI-assisted classification. Docsify runs as static frontend served by the Go server.

**Tech Stack:** Go 1.22+, gorilla/mux (or chi), blackfriday (Markdown preview), Docsify 4.x

## Task 2: Implement Article Store (File Storage + Index)

**Objective:** Build the core storage layer that manages Markdown files on disk and maintains the JSON index.

**Files:**
- Create: `internal/server/store.go`
- Create: `internal/server/store_test.go`

**Step 1: Implement Store struct and methods**

The Store must:
- Store Markdown files under `<dataDir>/articles/<category>/<slug>.md`
- Maintain `index.json` at `<dataDir>/index.json` — an array of Article objects
- Provide CRUD operations: `Create`, `Get`, `List`, `Delete`
- Support versioning: when re-publishing, archive old version to `<dataDir>/articles/<category>/.versions/<slug>/`
- Auto-generate a URL-safe ID from title + date (e.g., `ai-agent-patterns-20260517`)
- Extract first 200 chars as summary if not provided
- Thread-safe index access via `sync.RWMutex`

Key methods:

```go
type Store struct {
    dataDir string
    mu      sync.RWMutex
    articles []model.Article
}

func New(dataDir string) (*Store, error)  // Load existing index.json or create empty
func (s *Store) Create(req model.PublishRequest) (*model.PublishResponse, error)
func (s *Store) Get(id string) (*model.Article, error)
func (s *Store) List(filter ListFilter) ([]model.Article, error)
func (s *Store) Delete(id string) error
func (s *Store) loadIndex() error
func (s *Store) saveIndex() error
func (s *Store) generateID(title string) string
func (s *Store) extractSummary(content string) string
func (s *Store) slugify(title string) string
```

**ListFilter:**

```go
type ListFilter struct {
    Category string
    Tag      string
    Author   string
    Query    string  // search in title/summary
}
```

**Version handling in Create:**
- If `req.VersionOf` is non-empty, find existing article by that ID
- Move current file to `.versions/<slug>/v<N>-<date>.md`
- Save `.versions/<slug>/meta.json` with version history
- Increment version number
- Save new file in place

## Task 4: Implement HTTP API Handlers

**Objective:** Expose REST API for article CRUD and serve Docsify static files.

**Files:**
- Create: `internal/server/handler.go`
- Create: `internal/server/handler_test.go`

**Step 1: Set up router and handlers**

Use Go standard library `http.ServeMux` (no external dependency needed for Go 1.22+):

```go
func (s *Store) Handler() http.Handler {
    mux := http.NewServeMux()

    // API routes
    mux.HandleFunc("POST /api/articles", s.handleCreateArticle)
    mux.HandleFunc("GET /api/articles", s.handleListArticles)
    mux.HandleFunc("GET /api/articles/{id}", s.handleGetArticle)
    mux.HandleFunc("DELETE /api/articles/{id}", s.handleDeleteArticle)

    // Static files (Docsify + articles)
    mux.Handle("/", http.FileServer(http.Dir(s.dataDir)))

    return mux
}
```

**Step 2: Implement each handler**

- `handleCreateArticle`: Parse JSON body → `PublishRequest` → call `Store.Create` → return `PublishResponse` as JSON (201)
- `handleListArticles`: Parse query params (category, tag, author, q) → call `Store.List` → return article array as JSON (200)
- `handleGetArticle`: Extract `id` from path → call `Store.Get` → return article as JSON (200) or 404
- `handleDeleteArticle`: Extract `id` from path → call `Store.Delete` → return 204 or 404

Error responses use JSON: `{"error": "message"}`

**Step 3: Add CORS middleware for LAN access**

```go
func corsMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.Header().Set("Access-Control-Allow-Origin", "*")
        w.Header().Set("Access-Control-Allow-Methods", "GET, POST, DELETE, OPTIONS")
        w.Header().Set("Access-Control-Allow-Headers", "Content-Type")
        if r.Method == "OPTIONS" {
            w.WriteHeader(http.StatusOK)
            return
        }
        next.ServeHTTP(w, r)
    })
}
```


Flow:
1. Read the Markdown file
2. Parse frontmatter (if present) for title, tags, category
3. If `--classify` provided, use those values
4. If `--category`/`--tags` provided, use those
5. Otherwise, display AI suggestion placeholder (for Phase 2)
6. Show confirmation prompt unless `--yes`
7. POST to server `/api/articles`
8. Print result

Frontmatter parsing — support this format at the top of .md files:

```markdown
---
title: Article Title
category: AI
tags: [llm, rag, workflow]
author: radial
---
```

Use `gormetheus/yaml` or simple regex for parsing.
